    CSS animations are animations that changes elements from one style to another. The change can include as many properties as you want, and the change can occur many times as well. To use animations in CSS you must use something called keyframes. keyframes are styles that the elements will hold at certain times. The keyframes rule will implement a change in style gradually from one style to another and to make that work you have to bind the animation to an element. Below is an example of how the code should look like for keyframes:

    /* Code for animation */
    @keyframes animate {
     from {background-color: green;}
     to {background-color: blue;}
    }
    /* Element to bind animation */
    Div {
     background-color: green;
     animation-name: animate;
     animation-duration: 4s;
    }
    Animation example of a square changing colors:

    There are other sub properties under the animation property. Some of them include: animation-delay; this impacts the time at which the element will load and the start of an animation, animation-direction; this determines whether an animation should be forward or backwards, animation-duration; this determines the length of time for an animation to complete one cycle, animation-iteration-count; this is used to specify the number of times the animation will occur, and many more!
    You can also set multiple animation properties values. These properties are separated through commas and will apply multiple animations in a single rule. In this code example below, the fadeInOut has a duration of 3s and iterates 2 times and the bounce animation has a duration of 1s and will iterate 4 times:

    animation-name: fadeInOut, moveLeft300px, bounce;
    animation-duration: 3s, 5s, 1s;
    animation-iteration-count: 2, 1, 4;

    With CSS animations you have all the power in your hands. You can create whatever you want and whatever you like. Alongside CSS, you can use JavaScript to create more interactive animations and a more pleasing experience on a webpage. There are several pros and cons of each: CSS animations can be used for simpler transitions like if you want to toggle UI element states while JavaScript animations can be used when you want to have more advanced effects like slowing down, rewinding, stop and pause, etc. JavaScript can give you significant control over animations if you’re taking that approach and is ideal for super complex object-oriented applications. However, that doesn’t mean CSS animations are not good enough. Pure CSS also creates pleasing designs for animation. All the power is in the hands of the developer!
